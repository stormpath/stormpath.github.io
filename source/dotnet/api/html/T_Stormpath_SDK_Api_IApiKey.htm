IApiKey Interface

Remarks
An Account may have zero or more API Keys, and allowing multiple keys is often useful for key rotation strategies. For example, a new key can be generated while an existing key is in use. Applications can then reference the new key (e.g., on startup), and once running, the old key can then be deleted. This allows for key rotation without an interruption in service, which would happen otherwise if an old key was invalidated the instant a new key was generated.
