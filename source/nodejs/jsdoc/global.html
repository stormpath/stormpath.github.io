CacheOptions

Instances of Client will cache Stormpath resources in your local server, and this object allows you to configure the behavior of that cache. Pass this object to the Client constructor as clientOptions.cacheOptions. Examples are below the options table.

Type:
  • Object
Properties:
Name Type Argument Default Description
connection Object | String | Array <optional>

If you are allowing this library to create a Redis or Memcached client for you (you are specifying store but not providing a client), then use this option to provide the database connection information to the relevant library.

  • For Memcached, this can be any format supported by Memcached Sever Locations.

  • For Redis, provide an object with the host and port options.

Properties
Name Type Argument Default Description
host String <optional>
 localhost
port Number <optional>
 11211 | 6379
tti Number <optional>
 300

(Seconds) The idle time of a cached resource. If it is not accessed within this time, it will be purged.
ttl Number <optional>
 300

(Seconds) The max age of a cached resource. The resource will be purged after this time.
client Object <optional>

An existing Redis or Memcached client instance. If not provided, one will be created for you when you specify the store property.
options Object <optional>

If you are allowing this library to create a Redis or Memcached client for you (you are specifying store but not providing a client), then use this option to pass options to the relevant client constructor. For more information please see the documentation of those libraries:

  • Memcached options
  • Node-Redis options
store String <optional>

The type of cache store to use, can be redis or memcached. If not specified, an in-memory cache will be used for the duration of the Node process.
Source:
Client.js
Examples

Allow Stormpath to create a default Redis client for you:

var client = new stormpath.Client({
  cacheOptions: {
    store: 'redis'
  }
});

Specify the connection information for the default client:

var client = new stormpath.Client({
  cacheOptions: {
    store: 'redis',
    connection: {
      host: 'localhost',
      port: 7777
    }
  }
});

Provide your own Redis client:

var redisClient = redis.createClient();

var client = new stormpath.Client({
  cacheOptions: {
    store: 'redis',
    client: redisClient
  }
});

ClientOptions

This object allows you to configure the behavior of the Client and is provided when creating a new Client.

Type:
  • Object
Properties:
Name Type Argument Description
apiKey Object <optional>

An API Key Pair for your Tenant, see Create an API Key Pair.

Properties
Name Type Description
id String

The ID of the Tenant API Key Pair
secret String

The Secret of the Tenant API Key Pair
baseUrl String <optional>

The base URL for the Stormpath REST API. Enterprise Cloud customers should specify https://enterprise.stormpath.io/v1. Private deployments should use their custom base URL.
cacheOptions CacheOptions <optional>

Cache configuration, see CacheOptions.
nonceStore Object <optional>

If you are using the ID Site Feature in your Stormpath implementation, the calls to Application.createIdSiteUrl() and Application.handleIdSiteCallback() will make use of a nonce value to prevent replay attacks. By default these nonces will be stored in a cache region in the client's data store.

You may use your own Nonce Store by providing an interface object that we can use to communicate with it. The object should be passed as this nonceStore value, and it should have these two methods:

  • getNonce(nonceStringValue,callback) - It will search your nonce store for the nonce value and then call the callback with with the (err,value) pattern, where err indicates a problem with the store and value is the found nonce or null.
  • putNonce(nonceStringValue,callback) - It should place the nonce value in your nonce store and then call the callback with (err) where err is a store error or null.

CollectionQueryOptions

This object allows you to add query parameters when requesting collections from the Stormpath REST API. The query parameters allow you to sort, search, and limit the results that are returned. For more information, please see: REST API Reference: Collection Resources.

Type:
  • Object
Properties:
Name Type Argument Description
expand String <optional>

Comma-separated list of linked resources to expand on the returned resources. Not all links are available for expansion. For more information, please see: REST API Reference: Links.

Example: 'customData, directory'
limit Number <optional>

The maximum number of collection items to return for a single request. Minimum value is 1. Maximum value is 100. Default is 25.
offset Number <optional>

The zero-based starting index in the entire collection of the first item to return. Default is 0.
orderBy String <optional>

URL-encoded comma-separated list of ordering statements. Each ordering statement identifies a sortable attribute, and whether you would like the sorting to be ascending or descending.

Example: 'email,username asc'
q String <optional>

Stormpath will perform a case-insensitive matching query on all viewable attributes in all the resources in the Collection. Note that “viewable” means that the attribute can be viewed by the current caller. For full documentation of filter searching in the Stormpah API, please see: REST API Reference: Filter Search.
attribute String <optional>

Find objects by a specific attribute match, such as an email address. Not all attributes are available for matching on all resource types. Please refer to the specific resource type in this documentation, or see REST API Reference: Attribute Search.
Source:
resource/CollectionResource.js
Examples
// Expand custom data and directory when getting all accounts.
client.getAccounts({ expand: 'customData, directory' });
// Get all accounts, sorting by email, then username, ascending.
application.getAccounts({ orderBy: 'email,username asc' }, callbackFn);
// Find any account that has a searchable attribute containing the text “Joe”:
application.getAccounts({ q: 'Joe' }, callbackFn);
// Find all accounts with this email address.
application.getAccounts({ email: 'foo@bar.com'}, callbackFn);

ExpansionOptions

This object will pass the expand query parameter to the REST API when making requests for individual resources. The expand parameter allows you to fetch linked resources in the same request, allowing you to make less calls and warm your cache with the linked resources.

Type:
  • Object
Properties:
Name Type Argument Description
expand String <optional>

A comma-separated list of linked resources that should be expanded on the response from the REST API.
Source:
ds/DataStore.js
Example
client.getAccount({ expand: 'customData,groups' }, function(err, account){

  // The custom data object is now populated on the account object:

  console.log(account.customData)

  // The first page of the groups collection is available as well:

  console.log(account.groups.items)

  // But if you need to all the groups, you should use the
  // the getter method and iterate through the collection:

  account.getGroups(function(err, groupsCollection){
    groupsCollection.each(function(group){
      console.log(group);
    });
  });
});

Indicates that this account is a new account, and the registration was completed on ID Site.
status String

Indicates the action that the user completed on ID Site, which will be one of the following:

  • AUTHENTICATED - The user already had an account and they successfully authenticated ("logged in").
  • REGISTERED - The user created a new account on ID Site.
  • LOGOUT - The user has logged out.
