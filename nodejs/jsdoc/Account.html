Class: Account

Account

new Account(accountResource)

Encapsulates a Account resource. For full documentation of this resource, please see REST API Reference: Account.

For more information about managing your accounts with Stormpath, please see Account Management.

This class should not be manually constructed. It should be obtained from one of these methods:

  • Application.getAccount()
  • Application.getAccounts()
  • Client.getAccount()
  • Directory.getAccounts()
  • Group.getAccounts()
  • Organization.getAccounts()
  • Phone.getAccount()
  • Factor.getAccount()

addToGroup(group, callback)

Adds the account to the given group.

Parameters:
Name Type Description
group Object | String

If this parameter is a string, it should be the HREF of the group that the account will be added to.

If it is an object, it should have an href property which is the group href.
callback function

The function to call when the operation is complete. Will be called with the parameters (err, GroupMembership).
Source:
resource/Account.js
Examples
var groupHref = 'https://api.stormpath.com/v1/groups/xxx';
account.addToGroup(groupHref, callbackFn);
var group = {
  href: 'https://api.stormpath.com/v1/groups/xxx';
};

account.addToGroup(group, callbackFn);
var groupHref = 'https://api.stormpath.com/v1/groups/xxx';
client.getGroup(groupHref, function(err, group) {
  account.addToGroup(group, callbackFn);
});

createFactor(factor, callback)

Create a SmsFactor or GoogleAuthenticatorFactor for this account, which can be challenged for multi-factor authentication. For more information on this feature, please read Using Multi-Factor Authentication.

Parameters:
Name Type Description
factor Object

An object literal that describes the factor to create, the type must be specified. If this is an SMS factor, you can optionally describe the challenge, which will be automatically created and sent to the phone number.

Properties
Name Type Argument Description
challenge Object <optional>

An object literal that describes the Challenge object to automatically create.

Properties
Name Type Description
message String

The message for the challenge, which must include the placeholder ${code}.
phone Object <optional>

For SMS factors, an object literal that describes the Phone resource that will be created.

Properties
Name Type Description
number String

The phone number to send challenge codes to.
type String

Must be sms or google-authenticator.
callback function

The function to call when the operation is complete. Will be called with the parameters (err, SmsFactor) or (err, GoogleAuthenticatorFactor), depending on the specified type.
Source:
resource/Account.js
Examples

Creating an SMS Factor.

var factor = {
  type: 'sms',
  phone: {
    number: '+14151231234'
  }
};

account.createFactor(factor, function(err, smsFactor) {
  if (err) {
    return console.log(err);
  }

  console.log(smsFactor);
});

Creating a Google Authenticator factor.

var factor = {
  type: 'google-authenticator'
};

account.createFactor(factor, function(err, googleAuthenticatorFactor) {
  if (err) {
    return console.log(err);
  }

  console.log(googleAuthenticatorFactor);
});

getApiKeys(options)

Retrieves a collection of the account's API Keys.

The secret property of each resource is encrypted with the tenant API Key Secret that was used to configure the current Client instance. We do this to ensure that the secret is encrypted at rest, as it may be cached in your caching database.

Password-based AES 256 encryption is used. The PBKDF2 implementation will use 1024 iterations by default to derive the AES 256 key.

At the risk of potentially decreased security, you can use the encryptionKeySize option to specify a smaller encryption key size. You can also request a lower number of key iterations with the encryptionKeyIterations option. This can reduce the CPU time required to decrypt the key after transit or when retrieving from cache. It is not recommended to go much lower than 1024 (if at all) in security sensitive environments.

Parameters:
Name Type Description
options CollectionQueryOptions

Options for querying, paginating, and expanding the collection. This collection does not support filter searches. The following options can also be used with this request:

Properties
Name Type Argument Default Description
id String <optional>
 null

Search for a specific key by key id.
encryptionKeySize Number <optional>
 256

Set to 128 or 192 to change the AES key encryption size.
encryptionKeyIterations Number <optional>
 1024

Number of encryption iterations.
